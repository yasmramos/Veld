üìù Quick Example

Here's how simple dependency injection looks with Veld:

Basic Dependency Injection

import io.github.yasmramos.veld.annotation.Component;
import io.github.yasmramos.veld.annotation.Inject;
import io.github.yasmramos.veld.annotation.Singleton;

@Component
@Singleton
public class UserService {
    private final UserRepository userRepository;
    
    @Inject
    private UserRepository userRepository;
    
    public User findUser(String id) {
        return userRepository.findById(id);
    }
}

@Component
public class UserRepository {
    public User findById(String id) {
        // Database logic here
        return new User(id);
    }
}

üéØ Core Features

Dependency Injection

  • Constructor Injection: Automatic dependency resolution through constructors
  • Field Injection: Inject into private, static, and final fields
  • Method Injection: Support for setter and custom method injection
  • Provider Injection: Lazy and on-demand instance creation
  • Optional Injection: Graceful handling of missing dependencies

Scope Management

  • @Singleton: Single instance per application context
  • @Prototype: New instance for each injection
  • @Lazy: Deferred component creation
  • Custom Scopes: Define your own lifecycle patterns

Configuration & Value Injection

  • @Value: Inject from system properties, environment variables, and config files
  • Conditional Registration: @ConditionalOnProperty, @ConditionalOnClass
  • Profiles: Environment-based component activation
  • Configuration Classes: Centralized configuration management

Lifecycle Management

  • @PostConstruct: Initialization after dependency injection
  • @PreDestroy: Cleanup before bean destruction
  • Custom Lifecycle: Extend lifecycle with custom callbacks
  • Circular Dependency Detection: Compile-time detection with clear errors

üöÄ Advanced Features

Aspect-Oriented Programming (AOP)

Veld includes a powerful AOP framework built on ASM bytecode generation:

  • Dynamic Proxies: Generated at compile time for optimal performance
  • AspectJ-like Syntax: Familiar pointcut expression language
  • Multiple Advice Types: @Around, @Before, @After (RETURNING, THROWING, FINALLY)
  • CDI Interceptors: Full support for @Interceptor, @AroundInvoke
  • Built-in Interceptors: Logging, timing, validation, transactions

EventBus

Lightweight publish/subscribe event system:

  • Synchronous & Asynchronous: Choose event delivery mode
  • Priority Ordering: Control handler execution order
  • Type Hierarchy: Handlers receive events of declared type and subtypes
  • Weak References: Automatic cleanup of unsubscribed handlers

Java Platform Module System (JPMS)

Full JPMS support with strong encapsulation:

  • Module Descriptors: All modules include proper module-info.java
  • Strong Encapsulation: Clean module boundaries with explicit exports
  • Java 11-25 Compatible: Tested across all LTS versions

üöÄ Getting Started

Installation

Add Veld to your project with Maven or Gradle:

Maven

<dependencies>
    <!-- Veld Core -->
    <dependency>
        <groupId>io.github.yasmramos.veld</groupId>
        <artifactId>veld-runtime</artifactId>
        <version>1.0.2</version>
    </dependency>
    
    <!-- Veld Annotations -->
    <dependency>
        <groupId>io.github.yasmramos.veld</groupId>
        <artifactId>veld-annotations</artifactId>
        <version>1.0.2</version>
    </dependency>
</dependencies>

Gradle

dependencies {
    implementation 'io.github.yasmramos.veld:veld-runtime:1.0.2'
    implementation 'io.github.yasmramos.veld:veld-annotations:1.0.2'
}
