üöÄ Spring Boot Integration

Ultra-fast dependency injection for Spring Boot applications

Introduction

Veld provides seamless integration with Spring Boot, allowing you to leverage the ultra-fast compile-time dependency injection while maintaining full compatibility with existing Spring components.

Zero Configuration Required: Veld auto-configures itself when it detects Spring Boot on the classpath. No manual setup needed!

üì¶ Dependencies

Add Veld Spring Boot Starter

Maven 
<dependency>
    <groupId>io.github.yasmramos.veld</groupId>
    <artifactId>veld-spring-boot-starter</artifactId>
    <version>1.0.2</version>
</dependency>
Gradle 
implementation 'io.github.yasmramos.veld:veld-spring-boot-starter:1.0.2'
Automatic Setup: The starter includes all necessary dependencies and auto-configuration.

‚öôÔ∏è Auto-Configuration

Veld automatically configures itself when it detects Spring Boot. The starter provides:

  • Veld Container Initialization: Automatically starts the Veld container
  • Spring Boot Integration: Bridges Veld and Spring beans seamlessly
  • Health Checks: Integrates with Spring Boot Actuator
  • Configuration Properties: Exposes Veld settings via Spring Boot configuration

Configuration Properties

Customize Veld behavior using application.properties or application.yml:

application.properties 
# Enable/disable Veld
veld.enabled=true

# Enable Spring-Veld bean bridging
veld.spring-integration.enabled=true
veld.spring-integration.bridge-beans=true

# Veld configuration
veld.config-path=classpath:veld-config.properties
veld.scan-packages=com.yourpackage
application.yml 
veld:
  enabled: true
  spring-integration:
    enabled: true
    bridge-beans: true
  config-path: classpath:veld-config.properties
  scan-packages: com.yourpackage

üîÑ Bidirectional Bean Bridging

Veld provides seamless bidirectional integration between Veld and Spring containers:

Spring Beans in Veld

Spring beans annotated with @Component, @Service, or @Repository are automatically available in Veld:

SpringService.java 
@Service
public class SpringService {
    public void springMethod() {
        System.out.println("This is a Spring bean");
    }
}
VeldComponent.java 
@Component
public class VeldComponent {
    
    @Inject
    private SpringService springService; // Spring bean injected into Veld
    
    public void useSpringService() {
        springService.springMethod();
    }
}

Veld Beans in Spring

Veld components are automatically registered as Spring beans:

SpringComponent.java 
@Component
public class SpringComponent {
    
    @Autowired
    private VeldComponent veldComponent; // Veld bean injected into Spring
    
    public void useVeldComponent() {
        veldComponent.useSpringService();
    }
}

üéØ Conditional Bean Configuration

Use Veld's conditional annotations to control Spring bean creation:

ConditionalConfiguration.java 
@Configuration
public class ConditionalConfiguration {
    
    @Bean
    @ConditionalOnVeldBean(VeldComponent.class)
    public MyBean myBean(VeldComponent veldComponent) {
        return new MyBean(veldComponent);
    }
    
    @Bean
    @ConditionalOnVeldBean(value = VeldService.class, search = SearchStrategy.VELD_ONLY)
    public AnotherBean anotherBean(VeldService veldService) {
        return new AnotherBean(veldService);
    }
}

üîÑ Lifecycle Integration

Veld respects Spring Boot's lifecycle and integrates with:

  • ApplicationRunner: Veld container is fully initialized before ApplicationRunner runs
  • CommandLineRunner: Available for post-initialization tasks
  • Shutdown Hooks: Proper cleanup when Spring context shuts down
  • Health Checks: Exposes Veld health through Spring Boot Actuator

üîÑ Migration Guide

From Pure Spring Boot

Gradually migrate to Veld while maintaining existing Spring functionality:

Step 1: Add Dependencies 
<!-- Keep existing Spring Boot dependencies -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter</artifactId>
</dependency>

<!-- Add Veld -->
<dependency>
    <groupId>io.github.yasmramos.veld</groupId>
    <artifactId>veld-spring-boot-starter</artifactId>
    <version>1.0.2</version>
</dependency>
Step 2: Annotate Existing Components 
@Service // Keep @Service for Spring
public class UserService {
    @Inject // Use Veld @Inject alongside Spring @Autowired
    private UserRepository userRepository;
    
    // Veld will automatically bridge this to Spring
}

üêõ Troubleshooting

Common Issues

Veld not initializing

Solution: Check that veld.enabled=true in your configuration and that the Veld starter is on the classpath.

Beans not bridging

Solution: Ensure veld.spring-integration.enabled=true and that your beans are properly annotated.

Circular dependencies

Solution: Use @Lazy or restructure your dependencies to avoid cycles between Veld and Spring beans.
