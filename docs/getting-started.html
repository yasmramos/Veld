üöÄ Getting Started with Veld

üéØ Goal: By the end of this guide, you'll have a working Veld application with dependency injection, configuration, and lifecycle management.

üìã Prerequisites

System Requirements

  • Java: Version 11 or higher
  • Build Tool: Maven 3.6+ or Gradle 6+
  • IDE: IntelliJ IDEA (recommended), Eclipse, or VS Code
  • Memory: At least 512MB available for compilation
üí° Tip: Veld works with any Java build system and IDE. We provide examples for Maven and Gradle.

‚ö° Quick Installation

Maven Setup

Add Veld to your pom.xml:

<dependencies>
    <!-- Veld Core Runtime -->
    <dependency>
        <groupId>com.veld</groupId>
        <artifactId>veld-runtime</artifactId>
        <version>1.0.0-alpha.6</version>
    </dependency>
    
    <!-- Veld Annotations -->
    <dependency>
        <groupId>com.veld</groupId>
        <artifactId>veld-annotations</artifactId>
        <version>1.0.0-alpha.6</version>
    </dependency>
</dependencies>

<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.11.0</version>
            <configuration>
                <source>11</source>
                <target>11</target>
                <annotationProcessorPaths>
                    <path>
                        <groupId>com.veld</groupId>
                        <artifactId>veld-processor</artifactId>
                        <version>1.0.0-alpha.6</version>
                    </path>
                </annotationProcessorPaths>
            </configuration>
        </plugin>
    </plugins>
</build>

Gradle Setup

Add Veld to your build.gradle:

dependencies {
    implementation 'com.veld:veld-runtime:1.0.0-alpha.6'
    implementation 'com.veld:veld-annotations:1.0.0-alpha.6'
    annotationProcessor 'com.veld:veld-processor:1.0.0-alpha.6'
}

java {
    sourceCompatibility = JavaVersion.VERSION_11
    targetCompatibility = JavaVersion.VERSION_11
}

Step 1: Create a Component

Let's start by creating a simple service component:

package com.example.veld.service;

import com.veld.annotation.Component;
import com.veld.annotation.Singleton;

@Component
@Singleton
public class MessageService {
    
    public String getMessage() {
        return "Hello from Veld!";
    }
}

Step 2: Create a Configuration

Add configuration using @Value annotation:

package com.example.veld.config;

import com.veld.annotation.Component;
import com.veld.annotation.Value;

@Component
public class AppConfig {
    
    @Value("${app.name:VeldApp}")
    private String appName;
    
    @Value("${app.version:1.0.0}")
    private String version;
    
    public String getAppInfo() {
        return appName + " v" + version;
    }
}

Step 3: Create the Main Application

Bootstrap the Veld container:

package com.example.veld.main;

import com.veld.annotation.Component;
import com.veld.annotation.Inject;
import com.veld.runtime.Veld;
import com.example.veld.service.MessageService;
import com.example.veld.config.AppConfig;

@Component
public class Application {
    
    private final MessageService messageService;
    private final AppConfig appConfig;
    
    @Inject
    public Application(MessageService messageService, AppConfig appConfig) {
        this.messageService = messageService;
        this.appConfig = appConfig;
    }
    
    public void run() {
        System.out.println(appConfig.getAppInfo());
        System.out.println(messageService.getMessage());
    }
    
    public static void main(String[] args) {
        // Create and start Veld container
        // Use the generated Veld class;
        container.start();
        
        // Get the application bean and run it
        Application app = container.getBean(Application.class);
        app.run();
        
        // Shutdown the container
        container.shutdown();
    }
}

üíâ Dependency Injection Types

Veld supports multiple dependency injection patterns:

Constructor Injection (Recommended)

@Component
@Singleton
public class UserService {
    private final UserRepository userRepository;
    private final EmailService emailService;
    
    // Veld automatically injects dependencies
    @Inject
    public UserService(UserRepository userRepository, EmailService emailService) {
        this.userRepository = userRepository;
        this.emailService = emailService;
    }
}

Field Injection

@Component
@Singleton
public class OrderService {
    
    @Inject
    private UserRepository userRepository;
    
    @Inject
    private OrderRepository orderRepository;
}

Method Injection

@Component
@Singleton
public class NotificationService {
    
    private EmailService emailService;
    
    @Inject
    public void setEmailService(EmailService emailService) {
        this.emailService = emailService;
    }
}

Provider Injection (Lazy)

import com.veld.annotation.Provider;

@Component
@Singleton
public class ReportService {
    
    @Inject
    private Provider<ExpensiveService> expensiveServiceProvider;
    
    public void generateReport() {
        // ExpensiveService is created only when needed
        ExpensiveService service = expensiveServiceProvider.get();
        service.generateReport();
    }
}

üîÑ Scopes

Control the lifecycle of your components:

@Singleton (Default)

Single instance for the entire application:

@Component
@Singleton
public class DatabaseService {
    // Only one instance created
}

@Prototype

New instance for each injection:

@Component
@Prototype
public class RequestHandler {
    // New instance created each time it's injected
}

@Lazy

Defer component creation until first use:

@Component
@Lazy
public class ExpensiveService {
    // Created only when first accessed
}

‚öôÔ∏è Configuration

Veld provides flexible configuration management:

System Properties

@Component
public class AppConfig {
    
    @Value("${database.url}")
    private String databaseUrl;
    
    @Value("${database.username}")
    private String username;
}

Environment Variables

@Component
public class EnvConfig {
    
    @Value("${JAVA_HOME}")
    private String javaHome;
    
    @Value("${PATH}")
    private String path;
}

Configuration Files

Create application.properties in your resources:

app.name=MyVeldApp
app.version=1.0.0
database.url=jdbc:postgresql://localhost:5432/mydb
database.username=admin
üìù Note: Veld automatically loads application.properties and application.yml from your classpath.

üîÑ Lifecycle Callbacks

Hook into component lifecycle events:

@PostConstruct

Called after dependency injection is complete:

@Component
@Singleton
public class DatabaseService {
    
    @PostConstruct
    public void initialize() {
        // Database connection setup
        System.out.println("Database initialized");
    }
}

@PreDestroy

Called before bean destruction:

@Component
@Singleton
public class DatabaseService {
    
    @PreDestroy
    public void cleanup() {
        // Cleanup resources
        System.out.println("Database connection closed");
    }
}

Common Issues

  • Compilation Errors: Ensure annotation processor path is configured
  • Missing Dependencies: Check that all required components are annotated
  • Circular Dependencies: Veld detects these at compile time
  • Configuration Issues: Verify property files are in the classpath
