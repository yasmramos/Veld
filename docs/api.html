üèóÔ∏è Core API

Veld

Main container class for managing Veld components and dependency injection.

Constructor

public Veld()
public Veld(VeldConfiguration configuration)

Methods

Method Return Type Description
start() void Start the container and initialize all components
shutdown() void Shutdown the container and cleanup resources
<T> T getBean(Class<T> type) T Get a bean instance by type
<T> T getBean(String name, Class<T> type) T Get a bean instance by name and type
<T> List<T> getBeansOfType(Class<T> type) List<T> Get all beans of a specific type
boolean containsBean(String name) boolean Check if a bean with the given name exists
String[] getBeanNames() String[] Get names of all registered beans
void registerBean(Object bean) void Register a bean instance manually

Usage Example

// Use the generated Veld class;
container.start();

// Get beans
UserService userService = container.getBean(UserService.class);
List<UserRepository> repositories = container.getBeansOfType(UserRepository.class);

// Manual registration
container.registerBean(new CustomService());

container.shutdown();

VeldConfiguration

Configuration class for customizing Veld container behavior.

Builder Methods

Method Return Type Description
optimizationEnabled(boolean enabled) VeldConfiguration Enable/disable Phase 1 optimizations
cacheEnabled(boolean enabled) VeldConfiguration Enable/disable annotation cache
parallelEnabled(boolean enabled) VeldConfiguration Enable/disable parallel processing
threadPoolSize(int size) VeldConfiguration Set thread pool size for parallel processing
cacheSize(int size) VeldConfiguration Set annotation cache size
profile(String profile) VeldConfiguration Set active profile for conditional beans

Usage Example

VeldConfiguration config = VeldConfiguration.builder()
    .optimizationEnabled(true)
    .cacheEnabled(true)
    .parallelEnabled(true)
    .threadPoolSize(8)
    .cacheSize(10000)
    .profile("production")
    .build();

Veld container = new Veld(config);
container.start();

üì° EventBus API

EventBus

Main class for publish/subscribe event management.

Methods

Method Return Type Description
void publish(Object event) void Publish an event synchronously
void publishAsync(Object event) void Publish an event asynchronously
void subscribe(Object subscriber) void Subscribe an object to receive events
void unsubscribe(Object subscriber) void Unsubscribe an object from events
void clear() void Clear all subscriptions
int getSubscriberCount() int Get number of active subscribers

Usage Example

@Inject
private EventBus eventBus;

// Publish events
eventBus.publish(new UserRegisteredEvent("user123", "john@example.com"));
eventBus.publishAsync(new HeavyProcessingEvent());

// Subscribe/unsubscribe
eventBus.subscribe(myEventHandler);
eventBus.unsubscribe(myEventHandler);

üîß Utility Classes

ComponentInfo

Information about a Veld component.

Methods

Method Return Type Description
String getName() String Get component name
Class<?> getType() Class<?> Get component class
Scope getScope() Scope Get component scope
List<FieldInfo> getFields() List<FieldInfo> Get injected fields

Scope

Enumeration of available scopes.

Values

  • SINGLETON - Single instance per container
  • PROTOTYPE - New instance for each injection
  • LAZY - Deferred creation until first access
  • REQUEST - One instance per request (web applications)
  • SESSION - One instance per session (web applications)

FieldInfo

Information about an injected field.

Methods

Method Return Type Description
String getName() String Get field name
Class<?> getType() Class<?> Get field type
boolean isOptional() boolean Check if field is optional
boolean isRequired() boolean Check if field is required

üìù Annotations API

Core Annotations

All Veld annotations are located in the com.veld.annotation package.

Component Annotations

  • @Component - Define a managed component
  • @Module - Define a configuration module
  • @Primary - Mark component as primary
  • @Order - Define component ordering

Injection Annotations

  • @Inject - Mark field/method/constructor for injection
  • @Optional - Mark injection as optional
  • @Provider - Inject a provider for lazy creation
  • @Value - Inject configuration values

Scope Annotations

  • @Singleton - Single instance scope
  • @Prototype - Multiple instance scope
  • @Lazy - Lazy initialization

Lifecycle Annotations

  • @PostConstruct - Post-initialization callback
  • @PreDestroy - Pre-destruction callback

AOP Annotations

  • @Aspect - Define an aspect
  • @Pointcut - Define a pointcut
  • @Before - Before advice
  • @After - After advice
  • @Around - Around advice

Event Annotations

  • @Subscribe - Mark method as event handler

‚öôÔ∏è Configuration API

Configuration Properties

Veld can be configured using system properties, environment variables, or configuration files.

Core Properties

Property Type Default Description
veld.optimization.enabled boolean true Enable optimizations
veld.cache.enabled boolean true Enable caching
veld.parallel.enabled boolean true Enable parallel processing
veld.thread.pool.size int CPU_CORES Thread pool size
veld.cache.size int 1000 Cache size limit
veld.profile String default Active profile

Usage

# System properties
java -Dveld.optimization.enabled=true -Dveld.cache.size=10000 MyApp

# Environment variables
export VELD_OPTIMIZATION_ENABLED=true
export VELD_CACHE_SIZE=10000

# Configuration file (application.properties)
veld.optimization.enabled=true
veld.cache.size=10000
veld.profile=production
